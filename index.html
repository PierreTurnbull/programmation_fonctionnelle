Introduction à la programmation fonctionnelle

La programmation fonctionnelle est un paradigme de programmation avec des techniques particulières.

Dans les langages de programmation, on trouve aussi bien des langages de programmation purement fonctionnelle que des langages qui permettent l'utilisation de techniques de programmation fonctionnelle.

Haskell, Clojure et Scala font partie des langages de programmation purement fonctionnelle les plus populaires.

Les langages de programmation qui permmettent l'utilisation de techniques de programmation fonctionnelle sont JavaScript, Python, Ruby et bien d'autres.

La programmation fonctionnelle n'est pas un concept récent. Enfait, ses origines remontent aux années 1930, quand le calcul-lambda est né et a influencé de nombreux langages de programmation.

Ces derniers temps, elle gagne en popularité. C'est donc le moment idéal pour s'y intéresser.

Dans ce cours, je vais introduire les concepts principaux de la programmation fonctionnelle, en utilisant différents exemples en JavaScript.

Fonctions de première classe

Dans un langage de programmation fonctionnelle, les fonctions sont des objets de première classe.

Elles peuvent être affectées à des variables.

const f = (m) => console.log(m)
f('Test')

Etant donné qu'une fonction peut être affectée à une variable, elle peut être ajoutée à un objet:

const obj = {
  f(m) {
    console.log(m)
  }
}
obj.f('Test')

...ainsi qu'à une liste:

const a = [
  m => console.log(m)
]
a[0]('Test')

Elles peuvent être passées comme arguments à d'autres fonctions.

const f = (m) => () => console.log(m)
const f2 = (f3) => f3()
f2(f('Test'))

Elles peuvent être retournées par d'autres fonctions

const createF = () => {
  return (m) => console.log(m)
}
const f = createF()
f('Test')

Fonctions d'ordre supérieur

Les fonctions qui acceptent d'autres fonctions comme arguments ou qui retournent des fonctions sont appelées fonctions d'ordre supérieur.

Par exemple, dans la bibliothèque standard de JavaScript: Array.map(), Array.filter() et Array.reduce(), que nous verrons bientôt.

Programmation déclarative

Vous avez peut-être déjà entendu le terme "programmation déclarative".

L'inverse de la programmation déclarative est la programmation impérative.

Programmation déclarative vs impérative

Une approche impérative, c'est quand on dit à la machine les étapes qu'elle doit suivre afin de réaliser une tâche.

Une approche déclarative, c'est quand on dit à la machine ce qu'on doit faire et on la laisse décider des étapes.

Vous commencez à penser de manière déclarative quand vous avez un niveau d'abstraction suffisant pour arrêter de vous soucier des détails de bas niveau, afin de vous concentrer sur des choses plus générales (plus haut niveau).

Par exemple, le langage C est plus déclaratif qu'un langage assembleur.

Le JavaScript peut à la fois avoir une approche impérative et déclarative.

Par exemple, une approche déclarative serait d'utiliser des boucles et d'utiliser à la place des fonctions comme map, reduce et filter, car un programme construit ainsi aurait un plus haut niveau d'abstraction et serait moins focalisé sur dire à la machine toutes les étapes à suivre pour réaliser une action.

Immutabilité

En programmation fonctionnelle, la donnée ne change jamais: elle est immutable.

Plutôt que de modifier une variable, on en crée une copie qui reflète ce changement.

Un objet n'est jamais mis à jour, mais copié avant d'être changé.

const

C'est pourquoi const est tant utilisé en JavaScript moderne, qui embrasse les concepts de la programmation fonctionnelle: forcer l'immutabilité sur les variables.

Object.assign()

ES2015 introduit aussi Object.assign(), qui est un élément essentiel pour créer des objets:

const redObj = { color: 'red' }
const yellowObj = Object.assign({}, redObj, {color: 'yellow'})

concat()

Pour ajouter un élément à une liste, on a tendance à utiliser la méthode push(), mais cette méthode modifie la liste originelle, donc ce n'est pas de la programmation fonctionnelle.

Pour que ce soit de la programmation fonctionnelle, on utilise plutôt la méthode concat():

const a = [1, 2]
const b = [1, 2].concat(3)
// b = [1, 2, 3]

...ou on utilise le spread operator:

const c = [...a, 3]
// c = [1, 2, 3]

filter()

Il en va de même pour supprimer un élément d'une liste: plutôt que d'utiliser pop() et splice(), qui modifient la liste, on utilise array.filter():

const d = a.filter((v, k) => k < 1)
// d = [1]

Pureté

Une fonction pure:

  • ne change jamais aucun des paramètres qui lui sont passés par référence (objets et listes): ils devraient être considérés comme immutables. Elle peut cependant modifier les paramètres passés par valeur (booléens, chaînes de caractères, nombres, undefined, null)
  • la valeur de retour d'une fonction pure n'est influencée par rien d'autre que par les paramètres qui lui sont passés: si les paramètres passés à la fonction sont les mêmes, le résultat sera toujours le même.
  • Pendant son exécution, une fonction pure ne change jamais rien qui soit en dehors de sa portée.

Transformations de données

Etant donné que l'immutabilité est un concept essentiel de la programmation fonctionnelle, vous pourriez vous demander comment la donnée peut changer.

C'est simple: la donnée est "changée" en créant des copies et en modifiant ces copies. La donnée originelle n'est jamais changée.

Les fonctions, en particulier, changent la donnée en retournant des copies modifiées de celle-ci.

Les principales fonctions qui font ça sont map et reduce.

Array.map()

Appeler Array.map() sur une liste crée une nouvelle liste. Cette liste contient le résultat d'une même modification appliquée à tous ses éléments:

const a = [1, 2, 3]
const b = a.map((v, k) => v * k)
// b = [0, 2, 6]

Array.reduce()

Appeler Array.reduce() sur une liste nous permet de récupérer le résultat d'une même opération appliquée de manière cumulative sur tous ses éléments.

Il suffit de passer comme paramètre une fonction qui calcule le résultat, et un point de départ:

const a = [1, 2, 3]
const sum = a.reduce((partial, v) => partial + v, 0)
// sum = 6

const o = a.reduce((obj, k) => { obj[k] = k; return obj }, {})
// o = {1: 1, 2: 2, 3: 3}

Recursivité

La récursivité est un sujet essentiel en programmation fonctionnelle. Quand une fonction s'appelle elle-même, elle est dite récursive.

L'exemple classique de la récursivité est la suite de Fibonacci (N = (N-1 + N-2)):

var f = (n) => n <= 1 ? 1 : f(n-1) + f(n-2)

Enchaînement de fonctions

L'enchaînement de fonctions consiste à utiliser des fonctions d'ordre supérieur les unes à la suite des autres, en faisant utiliser le résultat de l'une par la suivante.

Enchaîner des fonctions en JavaScript pur

Chaînage

Si on connaît le type du résultat d'une fonction, on peut utiliser une fonction propre au type retourné:

obj.doSomething()
   .doSomethingElse()

Composition

On peut aussi passer le résultat d'une fonction comme argument à une autre fonction:

obj.doSomething(doThis())

Enchaîner des fonctions avec l'aide de lodash

De manière plus générale, l'enchaînement de fonctions est l'acte de mettre en place une liste de fonctions simples afin d'effectuer une opération plus complexe.

lodash/fp propose une implémentation de l'enchaînement de fonctions: on exécute une liste de fonctions en commençant par un argument, chaque fonction hérite de la valeur de retour de la précédent fonction comme argument. Notez qu'on ne range pas les valeurs intermédiaires n'importe où.

import { compose } from 'lodash/fp'

const slugify = compose(
  encodeURIComponent,
  join('-'),
  map(toLowerCase),
  split(' ')
)

slufigy('Hello World') // hello-world
